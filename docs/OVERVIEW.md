## 1 行で言うと

個人開発者および小規模開発チーム(2〜10 人)向けの、「アイデア → 設計 → 開発 → リリース → 初期ユーザー獲得」までを一元管理する **AI 支援付き B2B SaaS** です。

## 解決する課題

個人開発者および小規模チームがリリースまでに直面する 4 つの壁:

1. **設計の言語化が後回しになる**: README / LP / 告知文を毎回ゼロから書くため時間がかかり、作る前に消耗する
2. **リリース前のチェックが属人化する**: 技術 / 法務 / マーケ / UX の観点で見落としが起きやすい
3. **過去のプロジェクト経験が死蔵される**: 過去に書いたドキュメント・トラブル対処・告知文が再利用されない
4. **マーケ・法務・UX 等の非技術領域に手が回らない**: エンジニアが本業の合間にこなすには負荷が大きい

## ターゲット

- **個人開発者**: 副業 / 独立を視野に複数プロジェクトを並列管理したいエンジニア
- **小規模開発チーム(2〜10 人)**: 共同で個人発プロジェクト・ハッカソン作品・スタートアップ初期プロダクトを進めるチーム

共通の課題: リリースまでの工程を効率化したい / 過去経験を資産化したい / AI で雑務を減らしたい。

## 提供価値

- **AI による自動ドラフト**: 競合調査・LP・README・告知文を Anthropic Claude で生成
- **過去資産の RAG 検索**: 過去のプロジェクトドキュメントをベクトル検索し、新規生成時にコンテキスト注入(独自性の核)
- **リリース前チェックリスト自動生成**: 技術 / 法務 / マーケ / UX / その他の 5 カテゴリで自動生成
- **マルチプロジェクト並列管理**: 複数の個人開発を 1 つのワークスペースで一元管理
- **チーム機能(Team プラン)**: メンバー招待・6 種ロール(OWNER / ADMIN / DEVELOPER / REVIEWER / TESTER / VIEWER)・共同編集・レビュー・監査ログ

## 差別化要素

| 観点                       | Shipyard                          | 既存サービス                                 |
| -------------------------- | --------------------------------- | -------------------------------------------- |
| 個人開発リリース支援       | **メイン機能として位置付け**      | 部分的(ProductHunt は発見、Notion は汎用)    |
| 過去プロジェクトの RAG     | **独自機能(他に類似なし)**        | Notion AI / Mem.ai / Reor は組織知見の対象外 |
| マルチテナント + 課金 + AI | フルセット                        | 個別ツールの組み合わせが必要                 |
| ターゲットの絞り込み       | **個人開発者 + 小規模チーム特化** | 大企業向け or 汎用ツールが大半               |

レッドオーシャン領域(ナレッジ管理 / タスク管理 / カレンダー / メモ)は意図的に避け、「個人開発リリース支援」というニッチを掘っています。

## ビジネスモデル(ADR-012)

| プラン   | 月額        | メンバー | AI クレジット/月         | 主要機能                                              |
| -------- | ----------- | -------- | ------------------------ | ----------------------------------------------------- |
| 新規登録 | ¥0          | 1 人     | 300 cr(7 日トライアル中) | Pro 全機能を 7 日間(クレカ不要)                       |
| Pro      | ¥1,480 / 月 | 1 人     | 300 cr                   | Sonnet 4 / Haiku 4.5 自由切替、優先サポート           |
| Team     | ¥2,800 / 人 | 2 人以上 | 800 cr / 人(共有プール)  | + メンバー招待、共同編集、レビュー、監査ログ          |
| (Free)   | ¥0          | 1 人     | 0(AI 停止)               | トライアル終了 / 解約後のフォールバック(閲覧のみ可能) |

- **AI クレジット**: モデル別の従量制(Haiku 4.5 = 1 cr / Sonnet 4 = 3 cr / 月初リセット)
- **決済**: Stripe Subscriptions(Checkout リダイレクト型 + Webhook)
- **人数課金**: Subscription Quantity で Team プランのメンバー数を表現
- **解約フロー**: 7 日 grace → 30 日凍結 → 削除

詳細は [`docs/adr/004-billing-plans.md`](./adr/004-billing-plans.md)(課金基盤 / Stripe 連携)+ [`docs/adr/012-plan-structure-revision.md`](./adr/012-plan-structure-revision.md)(プラン構造改訂) を参照。

## 技術スタック(要約)

| 領域           | 採用                                                         |
| -------------- | ------------------------------------------------------------ |
| フロントエンド | Next.js (App Router) + TypeScript + Tailwind CSS + shadcn/ui |
| バックエンド   | NestJS + Prisma                                              |
| データベース   | PostgreSQL 16 + pgvector(RAG 用ベクトル検索)                 |
| 認証           | Clerk                                                        |
| 決済           | Stripe Subscriptions                                         |
| AI             | Anthropic API(Claude Sonnet 4 + Haiku 4.5 の使い分け)        |
| キュー         | Redis(ElastiCache)+ BullMQ                                   |
| インフラ       | Vercel(Web)+ AWS ECS Fargate(API)+ RDS Aurora Serverless v2  |
| Monorepo       | Turborepo + pnpm Workspaces                                  |

選定理由・棄却した代替は [`docs/adr/001-tech-stack.md`](./adr/001-tech-stack.md) に詳述。

## アーキテクチャの主要決定

| 領域                   | 決定                                          | ADR     |
| ---------------------- | --------------------------------------------- | ------- |
| マルチテナント分離方式 | Pool model(共有 DB + `tenantId` 自動注入)     | ADR-002 |
| テナント解決方式       | サブパス方式 `shipyard.app/w/{slug}`          | ADR-003 |
| 課金構造               | 3 プラン + Stripe Webhook Idempotency         | ADR-004 |
| AI モデル使い分け      | Sonnet 4(品質)+ Haiku 4.5(構造化、コスト 1/3) | ADR-005 |

## ロードマップ

3 週間で MVP リリース、その後本業の合間(週 12 時間)で成長フェーズへ。

| 期間    | 内容                                                                                 |
| ------- | ------------------------------------------------------------------------------------ |
| Week 1  | 設計と基盤(ADR / データモデル / 開発環境 / 認証 / マルチテナント / Stripe / AI 基盤) |
| Week 2  | コア機能実装(プロジェクト CRUD / AI 機能 6 種 / RAG / 検索 UI)                       |
| Week 3  | 課金・本番化(Stripe Checkout / AWS デプロイ / LP / Zenn 記事 / 公開リリース)         |
| Week 4+ | 成長フェーズ(フィードバック対応 / GitHub 連携 / VS Code 拡張 / 告知活動)             |

詳細スケジュールは [`PROJECT_STATUS.md`](./PROJECT_STATUS.md) セクション 6 を参照。
